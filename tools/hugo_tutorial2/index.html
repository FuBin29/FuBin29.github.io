修改首页的文章样式

文章概要

原本的概要会截取显示非常多的原文内容，我希望能够精简的描述文章内容，同时可以自定义概要。使用下面的方法可以使这部分文字变成每篇博客开头自定义的参数 Description 。

首先找到文件 /themes/Hyde/layouts/index.html ，然后将对应部分替换为下面的代码。

<article class="post">
  <h1 class="post-title">
    <a href="{{ .Permalink }}">{{ .Title }}</a>
  </h1>
  <time datetime="{{ .Date.Format "2006-01-02T15:04:05Z0700" }}" class="post-date">{{ .Date.Format "2006-01-02 15:04:05" }}</time>
  {{ .Description }}
  <!--{{ if .Truncated }}-->
  <!--<div class="read-more-link">
    <a href="{{ .RelPermalink }}">Read More…</a>
  </div>-->
  <!--{{ end }}-->
</article>

这里面的 {{ .Description }} 就是指用文章的参数 Description 作为首页上显示的文章概要。

同时不希望显示 Read More… 链接按钮。

日期

上面代码的这部分 .Date.Format "2006-01-02 15:04:05" }} 指明了我希望的日期风格。

具体情况

按照上面的分类目标，第一级用 category ，第二级用 tag ，第三级就是文章了。

archetype

用于简化每次创建新文章时的参数定义

例如 default.md 中定义如下：

---
title: "{{ replace .Name "-" " " | title }}"
date: {{ .Date }}
draft: false
Description: ""
Tags: ["hugo", "Linux"]
Categories: ["工具与技术", "电影", "读书札记", "大学课程笔记"]
---

他是默认的archetype，

在生成本篇博客时，会自动填充到新建的文章头部，然后按照需要修改为

---
title: "Hugo配置教程（二）网页布局及文章分类"
date: 2020-09-22T10:00:35+08:00
draft: false
Description: "调整网页的侧边栏索引，文章分类，增加按类索引页面，调整网页的内容呈现细节"
Tags: ["hugo"]
Categories: ["工具与技术"]
---
